같은 문제가 반복되지 않도록, 그리고 회고/발표 자료로 활용하기 위함입니다.

## 작성 규칙

- 파일명: `YYYY-MM-DD-{kebab-case-제목}.md` (예: `2026-04-22-redis-connection-timeout.md`)
- 이슈 발생 날짜 기준 (해결 날짜 아님)
- 새 문서 추가 시 위 표에 한 줄 추가
- 같은 날 여러 건이면 `YYYY-MM-DD-제목` 그대로 (날짜 뒤 숫자 붙이지 않음)
- 가능하면 관련 PR/커밋 링크 포함

---

## 템플릿

새 트러블슈팅 문서 작성 시 아래를 복사해서 사용하세요.

```markdown
# YYYY-MM-DD 이슈 제목

## 증상
어떤 현상이 어디서 관찰됐는지
- 발생 환경 (local / dev / prod)
- 재현 방법

## 원인
근본 원인 분석

## 해결
어떻게 해결했는지
- 관련 PR: #번호
- 관련 커밋: `해시`

## 배운 점 / 재발 방지
- 다음에 같은 상황이 오면 어떻게 대응할지
- 코드/프로세스 개선 포인트
 ```
